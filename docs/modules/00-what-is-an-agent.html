AI Agent란 무엇인가

개념

당신은 이미 매일 AI agent를 쓰고 있다 — Claude Code 자체가 에이전트다. 파일을 읽고, 명령을 실행하고, 결과를 보고 다음 행동을 정한다. 이 과정의 목표는 LLM을 "질문하면 답하는 것"으로 쓰는 단계에서, 그것을 스스로 일하는 시스템(agent)으로 엮는 단계로 넘어가는 것이다. 먼저 LLM과 agent가 정확히 무엇이 다른지부터 짚자.

LLM — "텍스트 in → 텍스트 out" 함수

정의 — LLM (Large Language Model): 텍스트를 입력받아 그 다음에 올 법한 텍스트를 생성하는 모델. 한 번의 호출은 stateless다 — 이전 호출을 스스로 기억하지 못한다.

개발자에게 익숙한 말로 하면, LLM 한 번의 호출은 부작용 없는 순수 함수에 가깝다.

# 개념적으로 LLM 호출은 이런 함수다
def llm(prompt: str) -> str:
    ...  # 같은 입력이면 대체로 같은 성격의 출력, 내부에 기억은 없음
왜 중요한가? LLM이 "기억하는 것처럼" 보이는 이유는 우리가 매번 이전 대화 전체를 다시 입력으로 넣어주기 때문이다. 모델 안에 상태가 쌓이는 게 아니다. 이 사실이 뒤에서 다룰 memory·context window의 출발점이다.

Agent — LLM을 loop로 감싼 시스템

정의 — Agent: LLM(두뇌)에 tools(행동 수단), goal(목표), 그리고 loop(반복)를 붙여, "판단 → 행동 → 관찰"을 목표 달성까지 되풀이하는 시스템.

비유하자면, LLM이 함수라면 agent는 그 함수를 감싼 REPL / 이벤트 루프다. 루프가 매 회 함수를 부르고, 함수가 내놓은 "행동"을 실제로 실행해(부작용=tools) 그 결과를 다시 다음 입력으로 먹인다(피드백=관찰).

핵심 루프 미리보기

이 과정 전체에서 우리가 만들 것의 심장은 이 한 덩어리다(M3에서 진짜로 구현한다):

context = [goal]
while not done:
    action  = llm(context)          # 1) 판단: 다음에 뭘 할지 결정 (도구 호출 또는 최종 답)
    result  = run(action)           # 2) 행동: 우리가 그 도구를 실제로 실행
    context = context + observe(result)   # 3) 관찰: 결과를 다시 context에 누적
# done = 모델이 "더 부를 도구 없음(end_turn)"이라고 말할 때

에이전트의 구성요소

  • model (두뇌) — 무엇을 할지 판단하는 LLM. 우리는 Claude를 쓴다.
  • tools (손발) — 파일 읽기, 명령 실행, 검색처럼 모델이 "불러 달라"고 요청하는 함수.
  • memory / context (메모장) — 지금까지의 대화·관찰 기록. 매 호출에 다시 넣어준다.
  • loop / harness (심장박동) — 판단→행동→관찰을 돌리는 우리 코드.
  • goal & stop (목적·종료) — 무엇을 끝낼지, 언제 멈출지.

왜 지금 에이전트인가

  • tool calling: 모델이 "이 도구를 이 인자로 불러줘"라고 구조화해 요청할 수 있게 됐다(M2).
  • long context: 한 번에 넣을 수 있는 입력이 커져, 긴 작업의 맥락을 유지할 수 있다.
  • 추론 향상: 여러 단계를 스스로 계획·검증하는 능력이 실용 수준에 도달했다.
비유의 한계: "순수 함수"라는 비유는 직관엔 좋지만 완벽하진 않다 — LLM 출력은 확률적이라 같은 입력에도 매번 똑같지 않을 수 있다. "기억 없는 함수"라는 점만 가져가자.

한눈 정리

  • LLM = stateless 함수(텍스트 in → 텍스트 out). 기억은 우리가 매번 넣어주는 것.
  • Agent = LLM + tools + goal + loop. 판단→행동→관찰을 반복.
  • 구성요소: model·tools·memory·loop·goal. 우리는 이걸 Hermes로 직접 만든다.
  • 이번 모듈에선 아직 loop도 tools도 없이, "두뇌"인 첫 API 호출만 해본다.

직접 사용해보기

아래 코드를 직접 타이핑해 실행해 보세요. 전제: Python 3.10+, 그리고 Anthropic API 키.

0) 준비

pip install anthropic
export ANTHROPIC_API_KEY="sk-ant-...";  # 본인 키로 (코드에 하드코딩 금지)
키는 console.anthropic.com에서 발급한다. 호출마다 토큰만큼 소액 과금되니, 실습은 작은 max_tokens로 한다.

1) 첫 호출 — "두뇌"에게 물어보기

가장 기본인 messages.create를 한 번 호출한다.

# step1.py
import anthropic

client = anthropic.Anthropic()          # ANTHROPIC_API_KEY 환경변수 자동 사용

resp = client.messages.create(
    model="claude-sonnet-4-6",          # 코스 기본 모델 (cost-효율 + 고성능)
    max_tokens=512,
    messages=[
        {"role": "user", "content": "AI agent를 한 문장으로 정의해줘."}
    ],
)

# 응답은 '블록'들의 리스트. 텍스트 블록만 골라 출력.
for block in resp.content:
    if block.type == "text":
        print(block.text)
python step1.py
AI agent는 목표를 달성하기 위해 도구를 사용하며 스스로 판단과 행동을 반복하는 LLM 기반 시스템입니다.

2) "기억이 없다"를 직접 확인하기

LLM이 stateless라는 걸 눈으로 보자. 두 번의 독립된 호출을 한다. 두 번째에서 방금 한 말을 모른다.

# step2.py
import anthropic
client = anthropic.Anthropic()

def ask(text):
    r = client.messages.create(
        model="claude-sonnet-4-6", max_tokens=200,
        messages=[{"role": "user", "content": text}],
    )
    return "".join(b.text for b in r.content if b.type == "text")

print(ask("내 이름은 정구야. 기억해줘."))
print("---")
print(ask("방금 내가 말한 내 이름이 뭐였지?"))   # 두 번째 '독립' 호출
네, 정구님 기억하겠습니다!
---
죄송하지만 이전 대화 내용이 없어 이름을 알 수 없습니다. 알려주시면 …
왜 못 기억할까? 두 번째 messages.create에는 첫 번째 대화가 들어있지 않다. API는 stateless라, "기억"하게 하려면 이전 메시지를 history로 다시 넣어줘야 한다 — 그게 바로 다음 모듈(LLM 다루기)에서 할 일이다.
막히면: AuthenticationError(401) → ANTHROPIC_API_KEY 미설정/오타. NotFoundError(404) → 모델 ID 오타(claude-sonnet-4-6 그대로). 연결 오류 → 네트워크/프록시 확인.

Hermes 만들기

이번 단계: Hermes의 "두뇌"만 만든다 — 아직 loop도 tools도 없이, 단발 호출 래퍼. (개념↔코드: 위에서 본 llm(prompt) 함수 = 아래 ask().)

1) 프로젝트 뼈대

저장소의 clone-project/ 안에 다음 구조를 만든다.

clone-project/
├── .env.example          # ANTHROPIC_API_KEY=...
├── requirements.txt      # anthropic, python-dotenv
└── hermes/
    ├── __init__.py       # 빈 파일
    ├── config.py
    ├── llm.py
    └── __main__.py

2) hermes/config.py — 설정 한 곳

# hermes/config.py
MODEL = "claude-sonnet-4-6"   # 모델 ID는 여기 한 곳에서만 바꾼다
MAX_TOKENS = 1024

3) hermes/llm.py — 두뇌

# hermes/llm.py
import anthropic
from .config import MODEL, MAX_TOKENS

client = anthropic.Anthropic()   # ANTHROPIC_API_KEY 환경변수 사용

def ask(prompt: str) -> str:
    """단발 질문 → 답변 텍스트. 아직 기억도 도구도 없는 'Hermes의 두뇌'."""
    resp = client.messages.create(
        model=MODEL,
        max_tokens=MAX_TOKENS,
        messages=[{"role": "user", "content": prompt}],
    )
    return "".join(b.text for b in resp.content if b.type == "text")

4) hermes/__main__.py — 실행 진입점

# hermes/__main__.py
import sys
from .llm import ask

def main():
    prompt = " ".join(sys.argv[1:]) or "너는 누구야?"
    print(ask(prompt))

if __name__ == "__main__":
    main()

앞으로 도구를 쓰고 여러 단계를 스스로 처리하는 에이전트로 성장할 거예요.
