# 복습 자료 생성 (review-generation)

사용자가 **실제로 헷갈렸던 것** 중심의 리마인드 학습 자료를 만든다. 일반론 나열 금지.

## 먼저 읽을 것
- `docs/data/qa-log.jsonl` (사용자가 실제 던진 질문들 — 핵심 입력), `docs/data/curriculum.json` (완료 모듈), 해당 모듈 콘텐츠.

## 생성 절차
1. qa-log를 읽어 **태그/모듈별 빈도**를 집계 → 자주 물은 주제 = 약점 후보.
2. 완료 모듈별 **핵심 개념 재정리**(모듈당 3~5줄)로 뼈대를 만든다.
3. **자문자답 카드**: 사용자가 실제 물었던 질문을 카드로(질문 → 접어둔 답). 원문 질문을 그대로 살린다.
4. **약점 섹션**: 가장 자주/반복해 물은 주제를 짚고 보강 포인트 + 관련 모듈 링크.
5. **다음 추천**: 다음 모듈 또는 미완 주제.

## 출력 → `docs/reviews/review-{YYYYMMDD}.html`
- site-builder의 스타일(`docs/assets/style.css`)과 레이아웃을 재사용(헤더/내비 동일 톤).
- 자문자답 카드는 `<details><summary>질문</summary>답</details>`로 접기.
- `docs/index.html`/리뷰 인덱스에 새 복습 링크 추가(site-builder 협조).
- 날짜는 오케스트레이터가 준 현재 날짜를 파일명/제목에 사용(스크립트 생성 금지).

## 빈 로그 처리
- qa-log가 비었으면(질문 없음) "아직 기록된 질문이 없습니다 — 학습 중 궁금한 점을 물어보면 자동 기록됩니다"를 안내하고, 완료 모듈 요지만으로 가벼운 복습을 만든다.

## 출력 체크
- [ ] 실제 질문 기반 카드  [ ] 모듈별 핵심 재정리  [ ] 약점 집계  [ ] 다음 추천  [ ] index에 링크
